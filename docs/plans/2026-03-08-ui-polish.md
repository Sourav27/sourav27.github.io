**Step 2: Add OS-default theme logic to `App.tsx`**

Find where `data-theme` is set (likely a `useEffect` or static attribute). Replace with:

```tsx
// At top of App component (or in a useEffect at app root):
useEffect(() => {
  const mq = window.matchMedia('(prefers-color-scheme: dark)');
  const apply = (dark: boolean) =>
    document.documentElement.setAttribute('data-theme', dark ? 'dark' : 'light');
  apply(mq.matches);
  mq.addEventListener('change', (e) => apply(e.matches));
  return () => mq.removeEventListener('change', () => {});
}, []);
```

Also expose a `toggleTheme` function and current theme state so Navbar can control it. The simplest approach: lift theme state to App and pass `theme` + `toggleTheme` as props to Navbar, or use a module-level state. Use a React context if App already uses one; otherwise a simple useState + prop-drill is fine for 2 components.

```tsx
const [theme, setTheme] = useState<'dark' | 'light'>(() => {
  return window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light';
});

useEffect(() => {
  document.documentElement.setAttribute('data-theme', theme);
}, [theme]);

const toggleTheme = () => setTheme((t) => (t === 'dark' ? 'light' : 'dark'));
```

Pass `theme` and `toggleTheme` to `<Navbar theme={theme} onToggle={toggleTheme} />`.

## Task R-9: Gallery — touch-press color reveal for mobile

**Problem:** Gallery images use `grayscale group-hover:grayscale-0 group-hover:scale-105` which only fires on cursor hover. On touch devices, images stay permanently grayscale — the reveal effect is completely inaccessible.

**Approach:** Add `activeIdx` state to `Gallery`. Bind `onTouchStart` to set the active image and `onTouchEnd`/`onTouchCancel` to clear it after an 800ms delay (so the user sees the color before it fades back). Merge the active condition into the existing className logic alongside the hover classes.

**Files:**
- Modify: `src/components/Gallery.tsx`

**Step 1: Add `useState` import and `activeIdx` state**

At the top of the `Gallery` component:

```tsx
import { useState, useRef } from 'react';

export const Gallery = () => {
  const [activeIdx, setActiveIdx] = useState<number | null>(null);
  const timeoutRef = useRef<ReturnType<typeof setTimeout> | null>(null);

  const handleTouchStart = (idx: number) => {
    if (timeoutRef.current) clearTimeout(timeoutRef.current);
    setActiveIdx(idx);
  };

  const handleTouchEnd = () => {
    timeoutRef.current = setTimeout(() => setActiveIdx(null), 800);
  };
```

**Step 2: Apply touch handlers and active classes to the large image (gallery[0])**

Replace the large image `<motion.div>`:

```tsx
<motion.div
  className="relative aspect-[3/4] overflow-hidden group rounded-lg cursor-pointer"
  onTouchStart={() => handleTouchStart(0)}
  onTouchEnd={handleTouchEnd}
  onTouchCancel={handleTouchEnd}
  ...
>
  <img
    src={gallery[0].src}
    alt={gallery[0].alt}
    className={`w-full h-full object-cover transition-all duration-500 ease-out group-hover:scale-105 ${
      activeIdx === 0 ? 'grayscale-0 scale-105' : 'grayscale group-hover:grayscale-0'
    }`}
  />
  <div className={`absolute inset-0 bg-black/60 transition-opacity duration-500 flex flex-col items-center justify-center p-4 text-center ${
    activeIdx === 0 ? 'opacity-100' : 'opacity-0 group-hover:opacity-100'
  }`}>
    <p className="font-mono text-[13px] text-stone-100 mb-1">{gallery[0].camera}</p>
    <p className="font-mono text-[11px] text-stone-400">{gallery[0].lens}</p>
  </div>
</motion.div>
```
